## ğŸ§© Alur Sistem Kerja

1. **Inisialisasi Sistem**

   * ESP32 Server membuat *web dashboard* dan API endpoint untuk menerima data dari tiap ruangan.
   * Database sudah berisi daftar dosen, mata kuliah, dan ruangan.

2. **Input Data oleh Dosen**

   * Dosen membuka web form di **tablet ruangan** (alamat lokal, contoh: `http://esp32client1.local`).
   * Dosen memilih **Nama**, **Mata Kuliah**, dan **Ruangan** dari dropdown.
   * Dosen menekan tombol **IN** untuk menandakan mulai mengajar.

3. **Kirim Data ke Server**

   * ESP32 Client di ruangan mengirim data JSON via WiFi ke ESP32 Server:

     ```json
     {
       "ruangan": "Lab Komputer 1",
       "dosen": "Budi Santoso",
       "matkul": "Jaringan Komputer",
       "status": "IN"
     }
     ```

4. **Server Update Data**

   * ESP32 Server menyimpan data ke database lokal.
   * Web dashboard otomatis memperbarui tampilan (tanpa reload, pakai WebSocket atau refresh AJAX 2 detik).

5. **Dashboard Monitoring**

   * Monitor menampilkan status tiap ruangan:

     | Ruangan | Dosen        | Mata Kuliah       | Status    |
     | ------- | ------------ | ----------------- | --------- |
     | Lab 1   | Budi Santoso | Jaringan Komputer | Digunakan |
     | Lab 2   | -            | -                 | Kosong    |

6. **Setelah Selesai**

   * Dosen menekan tombol **OUT**, data dikirim ulang ke server â†’ status ruangan kembali **Kosong**.
